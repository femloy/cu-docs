cleanup.gml

Runs when your mod is disabled.
You should use this if your mod adds anything with code.

Your mod doesn't exist anymore when it's disabled, but that stuff lingers.
If you don't deal with that, the game could have a memory leak or even crash.

Custom sprites and objects are handled automatically, but it's good practice to delete them yourself.

Example

sprite_delete(global.spr_icon);
This deletes the sprite previously created in init.gml.
