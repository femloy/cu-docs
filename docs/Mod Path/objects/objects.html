objects

You can add custom objects with the events you need.

You can name your objects whatever you want, without spaces or special characters, except underscores.
The object name is the folder's name. So, the create event for obj_example would be at MOD_PATH/objects/obj_example/create.gml.

All of the possible events are in the obj_example folder.

Usage

instance_create(0, 0, obj_example);
This creates an instance of the custom obj_example object at the top left of the room.

__OBJECT

Instances of custom objects are actually just obj_mod_object, and they're identified with their __OBJECT variable.

That means you can't use object_index with them, so instead: 
__OBJECT.name // Gets the name of the object as a string
__OBJECT.path // Gets the full directory path of the object as a string

You also can't do with obj_example, but you can do this: 
with obj_mod_object
{
    if __OBJECT != obj_example
        continue;
    // Do something with every obj_example
}

And another example: 
if other.object_index != obj_example
// WRONG

if !(other.object_index == obj_mod_object && other.__OBJECT == obj_example)
// Unfortunately correct

No easier way to handle this I'm afraid. Mod loaders are hard.
Also I have no idea what happens if you change __OBJECT. So don't do that.
